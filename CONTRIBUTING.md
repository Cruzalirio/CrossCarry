# Contributing to CrossCarry

Thank you for your interest in contributing to the CrossCarry package! We welcome contributions from the community to improve functionality, usability, and documentation. Contributions can range from bug fixes and feature enhancements to new examples or graphical tools.

## Reporting Issues
- Use the **Issues** tab to report bugs, unexpected behavior, or inconsistencies.
- Please include a reproducible example whenever possible.
- Issues can also include suggestions for new features, such as improvements in sample size calculations, power analysis, or visualizations of results.

## Feature Requests
- Feature requests are encouraged, especially for:
  - Enhancing sample size or power calculation tools.
  - Extending graphical outputs and visualization functions.
  - Expanding flexibility in modeling complex carry-over effects.
- When submitting a request, provide context and a description of the intended use case.

## Pull Requests
- Fork the repository and create a separate branch for your changes.
- Follow existing coding style and naming conventions.
- Include examples, tests, and documentation where appropriate.
- Submit a pull request for review, clearly describing the changes or new features.

## Documentation
- Improvements to function documentation, vignettes, and examples are highly appreciated.
- Ensure examples are reproducible and clear to users of varying experience levels.

## Code of Conduct
- Please maintain respectful and constructive communication in all interactions.
- Contributions should follow the principles of collaboration and transparency.

We appreciate all contributions, whether big or small. Your feedback and suggestions help CrossCarry continue to evolve as a robust and user-friendly tool for the analysis of crossover designs.
